# Defy Client Collaboration Guide

Defy Client is a Minecraft client being built collaboratively by two friends, with Codex or other AI assistants helping with development.

## Repository

- GitHub organization: **Defynd-Productions** (displayed informally as Defy Productions)
- Repository: **Defy-Client**
- Expected remote: `https://github.com/Defynd-Productions/Defy-Client.git`

Before pulling, pushing, opening a pull request, or making other remote changes, verify the local Git remote and confirm that it points to the Defynd-Productions organization. The local checkout and the connected GitHub account should refer to the same repository.

## Working Together

- Use GitHub as the shared source of truth.
- Pull the latest changes before beginning work.
- Make focused changes and describe them clearly in commits and pull requests.
- Avoid overwriting or discarding another contributor's unfinished work.
- Check the current branch and working tree before editing, committing, pulling, or pushing.
- When two people are changing the same area, coordinate through branches or pull requests to reduce merge conflicts.

AI assistants should read this file before working in the repository, inspect the existing project before making assumptions, preserve unrelated changes, and verify their work before handing it back.
